HTML Forms

Form Element Code Example Notes Output
Create a Form <form action="/example.html" method="POST">

</form>		 The action attribute determines where the information is sent

The method attribute is assigned a HTTP verb that is included in the HTTP request
Text Input <input type="text" name="first-text-field" id="text-input" value="text">

<input type="password" name="first-password-field" id="password" value="password">

<input type="number" name="first-number-field" id="number" step="2" value="number"> 		The type attribute determines how it renders on the web page and what kind of data it can accept. type="text" renders a text field that users can type into. Setting type="password" replaces the user input with another character like an asterisk (*) or a dot (.). Using type="number" restricts user input to just numbers (and a few special characters). An additional step attribute can be included to create arrows inside the input field to increase or decrease by the value of the attribute.
The id attribute is necessary so that a label can be associated with the text input
The name attribute is used to access the value typed into the input field once the form is submitted
A value attribute can be assigned so that users have a pre-filled text field when they first see the rendered form



Adding a Label <label for="text-input">Enter text: </label>
<input type="text" name="first-text-field" id="text-input"> 		The value of the for attribute must match the value of the id attribute of the corresponding text input
Range Input <label for="volume">Volume Control </label>
<input type="range" name="range-field" id="volume" min="0" max="100" step="1"> 		Creates a slider which limits options for the user
Using type="range" creates a slider which limits options for the user
The min and max attributes can be used to assign minimum and maximum values
The step attribute contols how smooth and fluid the slider is

Textarea Element <textarea id="blog" name="blog" rows="5" cols="30">Default text</textarea> A textarea is used when more than one row of text will be inputted

The rows and cols attributes can be added to determine the amount of rows and columns for the <textarea>
Submit a Form <input type="submit" value="Send"> The value attribute determines the text shown on the button. If the value attribute is omitted, the default text, Submit, is used
Requiring an Input <input id="allergies" name="allergies" type="text" required> The required attribute is used when fields with a form are not optional, meaning there must be information provided before it can be submitted.

The styling of the requirement message will vary from browser to browser
Set a Minimum and Maximum for a Number Field <input id="guests" name="guests" type="number" min="1" max="4"> The min and max attributes are used to specify the minimum and maximum numerical values a user may submit.

A warning will appear if the user enters a number outside the acceptable range
Checking Text Length <input id="summary" name="summary" type="text" minlength="5" maxlength="250" required> The minlength and maxlength attributes are declared to set the minimum and maximum characters for a text field

A warning will appear if the user tries to sumit too few text characters. No warning message will appear if the user attempts to submit too many text characters, instead they won't even be able to input text characters over the maximum
Matching a Pattern <input id="payment" name="payment" type="text" required pattern="[0-9]{14,16}"> For cases when we want user input to follow specific guidelines, we use the pattern attribute and assign it a regular expression (regex)

A warning message will appear if the user tries to submit input that doesn't follow the regex

In the example, [0-9]{14,16}, requires the user to enter a 14-16 digit number
